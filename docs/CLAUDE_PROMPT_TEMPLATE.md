### 2. Markdown 內容文件 (content.md)
純 Markdown 格式，包含：
- 正文段落
- 標題層級清晰 (## 和 ###)
- **粗體** 強調重點
- 引言用 > 開頭
- 無任何 HTML 標籤或組件調用

結構範例：
```markdown
# 文章標題

開場段落...

## 第一大章節

段落內容...

### 小章節

更多內容...

> **重點提示**：放在這裡

## 第二大章節

...
```

## 🎨 設計風格要求

**顏色使用（嚴格遵循）：**
- 主色（藍）：#0066CC → Tailwind: bg-blue-100, text-blue-700
- 強調（橙）：#FF6B35 → Tailwind: bg-orange-100, text-orange-700  
- 成功（綠）：#10B981 → Tailwind: bg-green-100, text-green-700
- 警告（琥珀）：#F59E0B → Tailwind: bg-amber-100, text-amber-700
- 灰色系：bg-gray-50, bg-gray-100, text-gray-600, text-gray-500

**特殊組件類型定義：**

| 組件 | badge 類型 | 使用場景 |
|-----|----------|--------|
| 標籤 / 最高等級 | `best` | 推薦選項、主流選擇 |
| 警告 / 高度 | `coral` | 高強度、危險提示 |
| 特殊 / 低度 | `green` | 特殊應用、替代選項 |

**圖表和視覺化規則：**
- 進度條百分比：0-100（根據相對強度）
- 最高 95%（最強），最低 20%（最弱）
- 避免 50-50 平衡，要有明確高低

**表格規則：**
- 第一欄為主欄（粗體、左對齐）
- 最多 4 欄，超過則拆分多個表
- 行數 4-5 行最佳，不超過 8 行

## ✅ 驗證清單

輸出前檢查：
- [ ] 所有 badge 類型都在「best / coral / green」中
- [ ] 每個 spec 至少有 2 個 metrics
- [ ] FAQ 至少 5 題，最多 7 題
- [ ] 流程圖不超過 5 步
- [ ] 所有百分比是 5 的倍數 (0, 5, 10... 95, 100)
- [ ] 沒有任何 HTML 標籤或 Astro 語法
- [ ] Markdown 標題只用 ## 和 ### (不用 # 或 ####)
- [ ] 描述文字簡潔清楚，避免廢話
