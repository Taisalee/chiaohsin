## 🏗️ 網站架構概覽

本網站使用 **Astro** 框架建置，數據集中管理在一個 JSON 檔案中，方便維護更新。

### 技術架構
- **框架**：Astro + TailwindCSS + DaisyUI
- **數據管理**：集中式 JSON 配置檔
- **圖片存放**：`public/images/` 目錄

### 網站頁面結構
```
首頁 (/)
├── 公司介紹
│   ├── 特色簡介 (/about)
│   ├── 基本資料 (/about/info)
│   └── 聯絡資訊 (/contact)
├── 最新消息 (/news)
└── 商品介紹 (/products)
    └── 各產品詳細頁面 (/products/[id])
```

## 🔍 如何新增/修改 SEO 關鍵字

SEO（搜尋引擎最佳化）關鍵字能幫助客戶在 Google 等搜尋引擎上找到你的網站。

### 步驟 1：開啟 SEO 設定

打開檔案：`src/data/site-data.json`

找到 `"seo"` 區塊（約在第 672 行）：

```json
"seo": {
  "title": "喬新針織有限公司—棉紗/白龍/花龍/尼龍/沾膠/工作用手套",
  "description": "棉紗手套｜膠手套｜尼龍手套...",
  "keywords": [
    "棉紗手套",
    "尼龍手套",
    "工作手套",
    ...
  ]
}
```

### 步驟 2：修改 SEO 設定

#### 📝 修改網站標題（title）
```json
"title": "你的新標題文字"
```
> 建議：標題長度控制在 60 個字元以內

#### 📝 修改網站描述（description）
```json
"description": "你的網站描述文字，說明公司提供什麼產品和服務"
```
> 建議：描述長度控制在 160 個字元以內

#### 📝 新增 SEO 關鍵字
在 `"keywords"` 陣列中加入新的關鍵字：
```json
"keywords": [
  "棉紗手套",
  "尼龍手套",
  "工作手套",
  "你的新關鍵字"  // ← 在這裡加入新關鍵字
]
```

### 步驟 3：同時更新公司關鍵字

建議同時更新 `"company"` 區塊中的 `"keywords"`（約在第 48-56 行）：

```json
"company": {
  ...
  "keywords": [
    "棉紗手套",
    "尼龍手套",
    "你的新關鍵字"  // ← 保持與 SEO keywords 一致
  ]
}
```

### ✅ SEO 關鍵字選擇技巧

1. **選擇客戶會搜尋的詞彙**：例如「工作手套批發」、「棉紗手套台南」
2. **包含地區名稱**：如「台南工作手套」、「官田手套工廠」
3. **包含產品特性**：如「防滑手套」、「耐磨手套」
4. **考慮長尾關鍵字**：如「棉紗手套批發價格」

## 📦 如何新增產品

### 步驟 1：準備產品圖片

1. 將產品圖片檔案命名為：`product_l_[產品ID].jpg`
   - 例如：`product_l_1500001.jpg`
2. 圖片建議尺寸：800x800 像素
3. 將圖片放入 `public/images/` 資料夾

### 步驟 2：選擇產品分類

目前網站有以下產品分類：

| 分類 ID | 分類名稱 |
|---------|----------|
| `cotton-gloves` | 棉紗手套 |
| `rubber-gloves` | 膠手套 |
| `nylon-gloves` | 尼龍手套 |
| `white-dragon-gloves` | 白龍手套 |
| `flower-dragon-gloves` | 花龍手套 |
| `masks` | 各式口罩 |
| `packaging` | 包裝材料、膠帶 |
| `hardware` | 建築五金 |
| `bulk-bags` | 太空包 |
| `cleaning` | 清潔用品 |
| `special-gloves` | 特殊手套 |

### 步驟 3：新增產品資料

打開 `src/data/site-data.json`，找到 `"products"` 陣列（約在第 190 行），在最後一個產品後加入新產品：

```json
"products": [
  // ... 現有產品 ...
  {
    "id": "1500001",
    "name": "新產品名稱",
    "category": "cotton-gloves",
    "size": "F",
    "description": "產品詳細描述文字...",
    "image": "images/product_l_1500001.jpg"
  }
]
```

### 欄位說明

| 欄位 | 說明 | 範例 |
|------|------|------|
| `id` | 產品唯一識別碼（數字字串） | `"1500001"` |
| `name` | 產品名稱 | `"7針20兩棉紗"` |
| `category` | 分類 ID（見上方表格） | `"cotton-gloves"` |
| `size` | 尺寸規格 | `"S"`, `"M"`, `"L"`, `"F"`, `"S/M/L"` |
| `description` | 產品描述 | `"詳細說明文字..."` |
| `image` | 圖片路徑 | `"images/product_l_XXXXX.jpg"` |

### ⚠️ 注意事項

1. **ID 必須唯一**：不能與現有產品 ID 重複
2. **JSON 格式**：每個欄位都要用雙引號包住，欄位之間用逗號分隔
3. **最後一個產品**：最後一個產品物件後面**不要**加逗號

### 範例：完整的新產品資料

```json
{
  "id": "1500001",
  "name": "13針防切割手套",
  "category": "special-gloves",
  "size": "M/L/XL",
  "description": "採用高強度纖維編織，具備優異的防切割性能，適合機械加工、玻璃處理等需要手部保護的作業環境。",
  "image": "images/product_l_1500001.jpg"
}
```

## ✏️ 如何修改產品資訊

### 步驟 1：找到要修改的產品

打開 `src/data/site-data.json`，在 `"products"` 陣列中搜尋產品名稱或 ID。

> 💡 **小技巧**：使用編輯器的搜尋功能（Ctrl+F 或 Cmd+F）

### 步驟 2：修改資料

直接修改對應欄位的值即可。

**範例：修改產品描述**

找到產品：
```json
{
  "id": "1464759",
  "name": "7針20兩棉紗",
  "description": "舊的描述文字..."
}
```

修改為：
```json
{
  "id": "1464759",
  "name": "7針20兩棉紗",
  "description": "新的描述文字..."
}
```

### 刪除產品

如果要刪除產品，直接刪除整個產品物件（包含大括號 `{ }` 及內容），並確保刪除後的 JSON 格式正確。

## 📂 如何新增產品分類

### 步驟 1：找到 categories 區塊

打開 `src/data/site-data.json`，找到 `"categories"` 陣列（約在第 100 行）。

### 步驟 2：新增分類

在陣列最後加入新分類：

```json
"categories": [
  // ... 現有分類 ...
  {
    "id": "new-category",
    "oldId": "999999",
    "name": "新分類名稱",
    "description": "分類描述文字，說明這個分類包含什麼產品",
    "icon": "🧤",
    "color": "#FF6B6B"
  }
]
```

### 欄位說明

| 欄位 | 說明 | 範例 |
|------|------|------|
| `id` | 分類唯一識別碼（英文小寫加連字號） | `"new-category"` |
| `oldId` | 舊系統 ID（可自訂） | `"999999"` |
| `name` | 分類顯示名稱 | `"防護手套"` |
| `description` | 分類描述 | `"各式防護用途手套..."` |
| `icon` | 表情符號圖示 | `"🧤"`, `"✋"`, `"🔧"` |
| `color` | 主題色（十六進位色碼） | `"#FF6B6B"` |

### 色碼參考

| 顏色 | 色碼 |
|------|------|
| 紅色 | `#FF6B6B` |
| 橘色 | `#FF9F43` |
| 黃色 | `#FECA57` |

## 🔗 如何新增/修改導航選單

打開 `src/data/site-data.json`，找到 `"navigation"` 區塊（約在第 58 行）。

### 主導航選單結構

```json
"navigation": {
  "main": [
    {
      "title": "選單項目名稱",
      "url": "/頁面路徑",
      "children": [
        {
          "title": "子選單項目",
          "url": "/子頁面路徑"
        }
      ]
    }
  ]
}
```

### 新增主選單項目

```json
{
  "title": "新選單",
  "url": "/new-page",
  "children": []
}
```

### 新增子選單項目

```json
{
  "title": "公司介紹",
  "url": "/about",
  "children": [
    {
      "title": "特色簡介",
      "url": "/about"
    },
    {
      "title": "新的子選單",  // ← 新增這個
      "url": "/about/new"
    }
  ]
}
```

> ⚠️ **注意**：新增選單項目後，需要確保對應的頁面檔案存在於 `src/pages/` 中。

## 🖼️ 如何更換圖片

### 更換產品圖片

1. 新圖片命名為與原圖相同的檔名
2. 將新圖片放入 `public/images/` 資料夾覆蓋舊檔

**檔名格式**：`product_l_[產品ID].jpg`

### 更換 Banner 圖片

1. 準備新的 banner 圖片
2. 命名為 `banner.png`（或 `banner.jpg`）
3. 放入 `public/skin/` 資料夾覆蓋舊檔

### 更換公司 Logo

1. 準備新的 logo 圖片
2. 命名為 `corpimg.png`
3. 放入 `public/images/` 資料夾覆蓋舊檔

### 圖片規格建議

| 圖片類型 | 建議尺寸 | 格式 |
|----------|----------|------|
| 產品圖片 | 800x800 px | JPG |
| Banner | 1920x600 px | PNG, JPG |
| Logo | 300x100 px | PNG（透明背景） |

## 🚀 如何發布網站更新

### 本地預覽（測試修改結果）

在終端機執行：
```bash
npm run dev
```

然後在瀏覽器開啟 `http://localhost:4321` 查看效果。

### 建置網站

確認修改無誤後，執行：
```bash
npm run build
```

### 部署到 GitHub Pages

1. 確認所有修改已儲存
2. 執行 Git 指令推送更新：

```bash
git add .
git commit -m "更新網站內容"
git push
```

3. GitHub Actions 會自動部署，約 2-5 分鐘後生效

## ❓ 常見問題與排解

### Q1：修改後網站沒有變化？

**解決方法**：
1. 確認檔案已儲存
2. 重新整理瀏覽器（Ctrl+F5 強制重新載入）
3. 清除瀏覽器快取

### Q2：網站出現錯誤訊息？

**可能原因**：JSON 格式錯誤

**檢查方法**：
1. 確認所有字串都用雙引號 `"` 包住
2. 確認物件之間有逗號分隔 `,`
3. 確認最後一個項目後面沒有多餘的逗號
4. 使用 JSON 驗證工具檢查格式

**線上 JSON 驗證工具**：https://jsonlint.com/

### Q3：圖片無法顯示？

**檢查事項**：
1. 圖片是否放在正確資料夾（`public/images/`）
2. 檔名是否正確（包含副檔名）
3. `site-data.json` 中的路徑是否正確

### Q4：如何備份網站資料？

**建議**：定期備份 `src/data/site-data.json` 檔案

1. 複製檔案到安全位置
2. 或使用 Git 版本控制

### Q5：忘記產品 ID 格式？

**產品 ID 規則**：
- 格式：7 位數字字串
- 範例：`"1464759"`
- 必須唯一，不能重複

## 📝 附錄：JSON 格式快速參考

### 基本語法

```json
{
  "字串欄位": "這是文字內容",
  "數字欄位": 123,
  "布林欄位": true,
  "陣列欄位": ["項目1", "項目2", "項目3"],
  "物件欄位": {
    "子欄位1": "值1",
    "子欄位2": "值2"
  }
}
```

### 常見錯誤

❌ **錯誤：使用單引號**
```json
'錯誤的字串'
```

✅ **正確：使用雙引號**
```json
"正確的字串"
```

---

❌ **錯誤：最後有多餘逗號**
```json
{
  "name": "測試",
  "value": "123",  // ← 這個逗號會導致錯誤
}
```

✅ **正確：最後沒有逗號**
```json
{
  "name": "測試",
  "value": "123"
}
```

---

❌ **錯誤：遺漏引號**
```json
{
  name: "測試"  // ← 欄位名稱沒有引號
}
```

✅ **正確：欄位名稱有引號**
```json
{
  "name": "測試"
}
```
