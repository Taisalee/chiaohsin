# 專案說明

此專案為喬新針織的產品型錄網站，使用 Astro 框架建置。

## 產品上架 SOP

1. 使用者提供產品圖片
2. 判斷資料來源：
   - 有基本介紹文字 → 以使用者提供的為主，再用網路資料補充
   - 只有圖片 → 分析圖片辨識產品，全靠網路搜尋取得規格與用途
3. 草擬產品內容（名稱、分類、描述、特色、規格等）
4. 與使用者討論確認，等待同意後才動手
5. 複製圖片到 `public/images/`，寫入 `src/data/products.json`

## 最新消息內容方向

適合發布的內容類型：

1. **新品上架公告** - 新增產品、新規格或新重量的棉紗手套
2. **庫存／採購提醒** - 年末旺季備貨提醒、特定產品大量到貨通知
3. **公司動態** - 品質堅持、服務時間或聯絡方式更新
4. **優惠活動** - 大量訂購折扣、特定品項促銷
5. **產業應用資訊（輕量版）** - 季節或產業推薦的手套類型

待撰寫文章：
- 「2025 年度新品上架：點膠、沾膠手套規格全面更新」（與近期上架產品呼應）
- 「旺季備貨提醒：棉紗手套現貨充足，歡迎洽詢報價」（促進詢價）

## 最新消息新增 SOP

1. 在 `src/content/news/` 新建一個資料夾（以英文命名，例如 `new-product-2026`）
2. 資料夾內建立 `index.md`，frontmatter 格式如下：

```markdown
---
title: "文章標題"
description: "文章摘要，用於 SEO meta description"
publishDate: 2026-01-01
tags: ["標籤一", "標籤二"]
draft: false
---

文章內容（Markdown 格式）
```

3. 儲存後列表頁 `/news` 會自動出現新文章，個別文章網址為 `/news/<資料夾名稱>`
4. 不需修改任何頁面程式碼

## 知識筆記 SOP

開發過程中遇到的注意事項或知識點，用以下表格格式整理，方便複製到 Notion：

| 欄位 | 內容 |
|------|------|
| **標題** | 知識點標題 |
| **內容** | 詳細說明與重點 |
| **分類** | Select（如：Astro 框架、檔案結構、優化、產品上架等） |
| **建立日期** | 當天日期 |
| **相關檔案** | 涉及的檔案路徑 |
| **標籤** | Multi-select（如：開發指南、最佳實踐、常見問題等） |

### 知識筆記檔案管理原則

- `docs/notion-project-guide.md` 是基礎架構說明，**不追加新內容，保持原樣**
- 每次有結構性修改或新知識點，在 `docs/` **新增獨立的 `.md` 檔**，例如：
  - `docs/notion-seo-guide.md`
  - `docs/notion-product-upload-sop.md`
- 每份文件專注一個主題，方便複製到 Notion 對應頁面
- 使用者說「幫我整理成 Notion 筆記」時，新增對應檔案，不修改舊檔
